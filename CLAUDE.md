## 弱いモデルが犯しがちなミスと防止ルール

1. **push前の確認を怠る** → pushは即公開。push前に必ずローカルで表示確認(HTTPサーバー配信。file://は不可)。壊れた状態が公開されると検索流入ユーザーに直撃する
2. **rules.jsの規約データを推測で書く** → 必ず公式規約ページで確認し、確認日(YYYY-MM-DD)をデータに残す。二次情報・記憶からの記述は禁止。このサイトの信頼の根幹
3. **generateCreditにサイト別分岐を書く** → データ駆動設計。サイト追加はrules.jsへのエントリ追加のみでコード変更なしが正しい
4. **記事を追加してsitemap.xmlを更新し忘れる** → 記事追加時は①sitemap.xmlにURL追加 ②ピラー記事(articles/credit-rules.html)からのリンク追加 ③GA4タグが新ページに入っているか確認、の3点セット
5. **有料版(credit-ledger)への導線を壊す** → 無料版の存在理由は有料版への誘導。導線リンクの削除・変更はユーザー確認必須
